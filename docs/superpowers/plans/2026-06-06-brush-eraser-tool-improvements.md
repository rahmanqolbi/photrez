# Brush and Eraser Tool Improvements impl Plan
> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Photrez brush and eraser tools feel controllable and predictable by wiring real settings, cursor feedback, stroke rendering, shortcut resizing, and clear blocked-state feedback.

**Architecture:** Keep the MVP hot path in the existing SolidJS + TypeScript `DocumentEngine` + WebGL2 renderer stack. Tool settings live in editor UI state, pointer routing passes a typed paint settings payload, the 2D overlay renders the active stroke, and commit still syncs the resulting bitmap into `DocumentEngine` plus WebGL texture upload. Dirty-rect history is intentionally deferred because the current Photrez history model is snapshot-based and changing it would widen the blast radius.

**Tech Stack:** SolidJS signals/context, TypeScript strict mode, Canvas 2D overlay, `ImageBitmap`, WebGL2 texture upload, Vitest component/unit tests, Playwright smoke as final optional frontend check.

## Context7 Notes

Context7 was queried for SolidJS. The implementation must follow these patterns:

- Use `createSignal` for brush and eraser setting state.
- Access signals by calling them, for example `brushSize()`.
- Input controls should use Solid event handlers such as `onInput={(event) => ...}` and read `event.currentTarget.value`.
- Global keyboard listeners must be removed with `onCleanup`.
- Context values should expose accessors and setters explicitly, matching the existing `EditorContextValue` style.

## Pain Points To Solve

1. Users cannot trust whether eraser is at full strength because the current UI displays fixed values and the stroke code ignores settings.
2. Brush and eraser should not accidentally share unwanted size or hardness behavior.
3. The cursor preview should match the actual active size and hardness.
4. Soft edges should be generated by a consistent falloff model instead of being a visual label only.
5. Locked, hidden, or transparency-protected layers should explain why painting is blocked.
6. Size changes should be quick while staying local to the active paint tool.

## Data and Interaction Flow

```text
User selects Brush/Eraser
  -> activeTool signal changes
  -> BrushOptionBar reads active settings
  -> BrushCursorOverlay reads active settings

User edits Size/Hardness/Opacity
  -> setter clamps value
  -> cursor updates immediately
  -> next stroke uses updated settings

Pointer down on canvas
  -> useCanvasPointerTools.prepareToolContext()
  -> getActivePaintToolSettings(activeTool, state)
  -> input-handler commits history snapshot before mutation
  -> onPaintStroke(points, isEraser, settings)

Pointer move
  -> input-handler appends stroke points
  -> onPaintStroke receives full points and same active settings
  -> useBrushOverlay draws only new segment into overlay canvas

Pointer up
  -> input-handler sends final stroke points
  -> useCanvasPointerTools calls commitBrushStroke()
  -> useBrushOverlay creates ImageBitmap from overlay canvas
  -> engine.setLayerImageBitmap(layerId, bitmap)
  -> renderer.uploadImage(layerId, bitmap)
  -> scheduler.requestRender()
```

```typescript
export type PaintTool = "brush" | "eraser";

export interface PaintToolSettings {
  size: number;
  hardness: number;
  opacity: number;
}

export interface PaintToolState {
  brushSize: number;
  brushHardness: number;
  brushOpacity: number;
  eraserSize: number;
  eraserHardness: number;
  eraserOpacity: number;
}

export interface PaintEditableLayer {
  locked?: boolean;
  visible?: boolean;
  lockTransparency?: boolean;
}

export const MIN_PAINT_SIZE = 1;
export const MAX_PAINT_SIZE = 500;
export const PAINT_SIZE_STEP = 5;

export function clampPaintSize(value: number): number {
  if (!Number.isFinite(value)) return MIN_PAINT_SIZE;
  return Math.max(MIN_PAINT_SIZE, Math.min(MAX_PAINT_SIZE, Math.round(value)));
}

export function clampPaintPercent(value: number): number {
  if (!Number.isFinite(value)) return 1;
  return Math.max(0, Math.min(1, value));
}

export function clampPaintSettings(settings: PaintToolSettings): PaintToolSettings {
  return {
    size: clampPaintSize(settings.size),
    hardness: clampPaintPercent(settings.hardness),
    opacity: clampPaintPercent(settings.opacity),
  };
}

export function getActivePaintToolSettings(tool: string, state: PaintToolState): PaintToolSettings {
  if (tool === "eraser") {
    return clampPaintSettings({
      size: state.eraserSize,
      hardness: state.eraserHardness,
      opacity: state.eraserOpacity,
    });
  }

  return clampPaintSettings({
    size: state.brushSize,
    hardness: state.brushHardness,
    opacity: state.brushOpacity,
  });
}

export function adjustPaintSize(
  tool: string,
  state: PaintToolState,
  delta: number,
): Pick<PaintToolState, "brushSize" | "eraserSize"> {
  return {
    brushSize: tool === "brush" ? clampPaintSize(state.brushSize + delta) : state.brushSize,
    eraserSize: tool === "eraser" ? clampPaintSize(state.eraserSize + delta) : state.eraserSize,
  };
}

export function getPaintToolBlockReason(
  layer: PaintEditableLayer | null | undefined,
  isEraser: boolean,
): string | null {
  if (!layer) return "No editable layer selected";
  if (layer.locked) return "Layer locked";
  if (!layer.visible) return "Layer hidden";
  if (isEraser && layer.lockTransparency) return "Transparent pixels protected";
  return null;
}
```

```typescript
import type { PaintToolSettings } from "./brushToolState";

export interface StrokePoint {
  x: number;
  y: number;
}

export function getDabSpacing(size: number): number {
  return Math.max(1, Math.min(20, Math.round(size * 0.25)));
}

export function buildStrokeDabs(points: StrokePoint[], size: number): StrokePoint[] {
  if (points.length <= 1) return points.slice();

  const spacing = getDabSpacing(size);
  const dabs: StrokePoint[] = [points[0]];

  for (let i = 1; i < points.length; i += 1) {
    const prev = points[i - 1];
    const next = points[i];
    const dx = next.x - prev.x;
    const dy = next.y - prev.y;
    const distance = Math.hypot(dx, dy);
    const steps = Math.max(1, Math.ceil(distance / spacing));

    for (let step = 1; step <= steps; step += 1) {
      const t = step / steps;
      dabs.push({
        x: prev.x + dx * t,
        y: prev.y + dy * t,
      });
    }
  }

  return dabs;
}

export function renderPaintStrokeToContext(
  ctx: CanvasRenderingContext2D,
  points: StrokePoint[],
  settings: PaintToolSettings,
  color: string,
  isEraser: boolean,
): void {
  if (points.length === 0) return;

  const radius = settings.size / 2;
  const hardRadius = radius * settings.hardness;
  const dabs = buildStrokeDabs(points, settings.size);

  ctx.save();
  ctx.globalCompositeOperation = isEraser ? "destination-out" : "source-over";
  ctx.globalAlpha = settings.opacity;

  for (const dab of dabs) {
    const gradient = ctx.createRadialGradient(
      dab.x,
      dab.y,
      Math.max(0, hardRadius),
      dab.x,
      dab.y,
      radius,
    );
    gradient.addColorStop(0, color);
    gradient.addColorStop(settings.hardness, color);
    gradient.addColorStop(1, "rgba(0,0,0,0)");

    ctx.fillStyle = gradient;
    ctx.beginPath();
    ctx.arc(dab.x, dab.y, radius, 0, Math.PI * 2);
    ctx.fill();
  }

  ctx.restore();
}
```

## Risks and Mitigations

- **Risk:** Canvas 2D radial dabs may be slower for very large brushes.
  - **Mitigation:** Dab spacing is bounded at 20px and MVP brush max is 500px. If performance is poor, lower max size or increase spacing in a follow-up.

- **Risk:** Soft eraser opacity can still feel cumulative.
  - **Mitigation:** Use `Strength` wording and keep `Flow` out of MVP. Full hard erase is one click via `Hard 100`.

- **Risk:** Snapshot history remains heavy for large paint strokes.
  - **Mitigation:** Dirty-rect command history is deferred as a dedicated history refactor.

- **Risk:** Tests that mock `ToolContext` will fail after adding `paintSettings`.
  - **Mitigation:** Update all `ToolContext` fixtures in the same task that changes the type.
